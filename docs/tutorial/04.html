#4 : あたり判定

この章では、あたり判定の方法について解説していくよ。
まずは、この章のサンプルデモを紹介するね。

📥 script.txt 
void Main()
{
    Font font(60);
    int n = 100;
    int i;
    Array<Vec2> v(n);
    Array<Circle> b(n);

    for (i = 0; i < n; i++)
    {
        v[i] = RandomVec2(Scene::Width(), Scene::Height());
        b[i] = Circle(v[i], 5);
    }

    while (System::Update())
    {
        Circle o(Cursor::Pos(), 150);
        o.draw(ColorF(1.0, 0.1));

        for (i = 0; i < n; i++)
        {
            b[i].moveBy(v[i] * Scene::DeltaTime());

            if ((b[i].y < 0 && v[i].y < 0) || (Scene::Height() < b[i].y && v[i].y > 0))
            {
                v[i].y *= -1;
            }

            if ((b[i].x < 0 && v[i].x < 0) || (Scene::Width() < b[i].x && v[i].x > 0))
            {
                v[i].x *= -1;
            }

            if (b[i].intersects(o)) { b[i].draw(Palette::Yellow); }
            else { b[i].draw(Palette::Gray); }
        }

        font("Hello, SivBoard!🐣").drawAt(Scene::Center(), Palette::White);
    }
}

小さなボールがたくさん！
それと、半透明の大きな円があるね。

この大きな円にあたっているボールは黄色、
あたっていないボールは灰色にするというデモだよ。
上から順番に説明していくね。

#4-1 : ボールの生成

まず、たくさんのボールを生成するために、
今回は配列を使っているよ。
ボールは、100個用意しているんだ。

    int n = 100;
    Array<Vec2> v(n);
    Array<Circle> b(n);

Array というのが、配列なのかな？
v と b の2つの配列があるね。

そうだね。v はボールの移動量の配列で、
b はボールの描画で使う Circle の配列だよ。

#4-2 : 初期値の設定

次に、この配列に初期値を入れていくよ。
for 文でまとめて処理しているんだ。
for 文の書き方は、C言語と同じだね。

    int i;

    for (i = 0; i < n; i++)
    {
        v[i] = RandomVec2(Scene::Width(), Scene::Height());
        b[i] = Circle(v[i], 5);
    }

i が 0 ～ 99 までの100回分繰り返すのね。
RandomVec2() というのはなに？

ランダムな Vec2 の値を生成する関数だよ。
v にランダムな移動量を入れて、
その v で b の初期位置も決めているんだ。

#4-3 : あたり判定をする円の表示

次に、あたり判定をする円を用意するよ。
Cursor::Pos() は、現在のマウスの位置を取得できる。
つまりこの円は、マウスにあわせて動くんだ。

        Circle o(Cursor::Pos(), 150);
        o.draw(ColorF(1.0, 0.1));

あー、そういえばこの円は動かせるね。
ColorF() は色を指定しているの？

ColorF() は色を 0.0 ～ 1.0 で指定できる関数だよ。
基本は、ColorF(r, g, b, α) だけど、ここでの
パラメータは、RGBの色と、α値（透過）の2つだ。

#4-4 : ボールを動かす

次に、ボールを動かす部分を作るよ。
Scene::DeltaTime() は、直前のフレームからの経過時間。
この時間に v の移動量を掛けて、位置を動かしている。

        for (i = 0; i < n; i++)
        {
            b[i].moveBy(v[i] * Scene::DeltaTime());

            if ((b[i].y < 0 && v[i].y < 0) || (Scene::Height() < b[i].y && v[i].y > 0))
            {
                v[i].y *= -1;
            }

            if ((b[i].x < 0 && v[i].x < 0) || (Scene::Width() < b[i].x && v[i].x > 0))
            {
                v[i].x *= -1;
            }
        }

moveBy() という関数で動かしているのね。
その下の if 文では、なにをしているの？

ボールが画面から、はみ出ないようにしているんだ。
2つの if 文で、縦方向と横方向をチェックしている。
はみ出る場合は、移動量を反転させているんだ。

なるほどね～。

#4-5 : あたり判定

最後に、あたり判定だ。
あたっていれば黄色、あたっていなければ灰色。
intersects() という関数を使っているよ。

            if (b[i].intersects(o)) { b[i].draw(Palette::Yellow); }
            else { b[i].draw(Palette::Gray); }

え、これだけ？

うん。。シンプルでしょ。
ちなみに、Geometry2D::IntersectAt() という関数を使うと、
あたった部分の座標も取得できる。

肝心のあたり判定部分は、
これだけで作れちゃうんだね。

#4-X : 編集後記

配列については、データ構造の章で解説予定だよ。
ランダムを扱う関数は、ランダムの章で解説予定だよ。
Geometry2D::IntersectAt() は、実装上の都合で、
Siv3D と違って、処理が重くなるから注意してね。
