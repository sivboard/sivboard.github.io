SivBoard - Tutorial

#1 : SivBoardの基本

この章では、SivBoardの基本について解説していくよ。
まずはどうやって使うかの説明の前に、この章のサンプルデモを紹介するね。

📥 script.txt 
void Main()
{
    Font font(60);
    ParticleSystem2D ps(Scene::Center());

    CircleEmitter2D e;
    e.sourceRadius = Scene::Width();
    ps.setEmitter(e);

    ParticleSystem2DParameters p;
    p.startSize = 100;
    p.startLifeTime = 10;
    p.startAngularVelocityDeg = 100;

    while (System::Update())
    {
        ClearPrint();
        Print << Profiler::FPS() << " fps";

        p.startColor = HSV(360 * Periodic::Sawtooth0_1(60), 0.5, 1, 0.5);
        ps.setParameters(p);
        ps.update();
        ps.draw();

        font("Hello, SivBoard!🐣").drawAt(Scene::Center(), p.startColor);
    }
}

いやいや。。
いきなりハードルが高すぎじゃないですかね？

ごめんごめん、まずは雰囲気だけ掴んでもらえたらと思ってね。
基本は、上の様なスクリプトファイルをSivBoardに読み込ませて使うんだ。
テキストファイルだから、メモ帳などで編集できるよ。

よかった。。
最初で置いてけぼりになると思ったよ。

#1-1 : 基本画面

SivBoardを起動すると、下のような基本画面が表示されるよ。
ここで、F5キーを押すと、スクリプトファイルを読み込む画面がでてくる。
他に、F1キーでヘルプ表示、F12キーでSiv3Dのライセンスが表示できる。

じゃあ、さっきのスクリプトファイルをダウンロードして、
F5キーで読み込ませて、、おー、表示できた。60fpsで滑らかに動いてる。
毎回、こうやって読み込んで使う感じなの？

もう一つの使い方として、SivBoardと同じディレクトリに、
"script.as"の名前でファイルを置いておくと、起動時に自動で読み込んでくれる。
ファイルがなければ、基本画面で起動する。という作りになっているんだ。

#1-2 : ファイルを読み込んだ後の動作

あれ、スクリプトファイルを読み込んだ後だと、F1/F12キーが反応しない。。
F5キーも、さっきの様な読み込みの画面がでてこないんだけど。

それらはあくまでも、基本画面の時の機能なんだ。
読み込んだ後は、そのコードに従う。F5キーは、更新ボタンの役割になるけど、
これも、System::SetReloadKeyF5(false); で無効化が可能だ。

必要なら、自分で機能を追加できるってことね。
ファンクションキーを押した時の動作も作れるのか。

#1-3 : プログラミングコードの更新

さっきのスクリプトファイルのパラメータを変更したいんだけど、
更新するには、一度終了して再起動が必要なのかな？

更新は、いま読み込んでいるスクリプトファイルを
テキストエディタ等で編集して上書きすると、即座に反映されるよ。
試しに、11行目を p.startSize = 10; に変更してみて。

100 を 10 にすればいいのね、、ポチっと。
お、図形が小さくなった！編集結果がすぐに確認できて良いね。

#1-4 : スクリプトファイル

スクリプトファイルについても説明するよ。
ポイントは２つ。まず、基本的には void Main() に処理を書く。
そして、その中の while (System::Update()) には、ループ処理を書くんだ。

自分で作った関数を、呼び出すことはできないの？

同一ファイル内であれば、それもできるよ。
例えば、Draw() という関数を作って、Main() から呼び出す、とかね。

#1-5 : デバッグ表示

左上にフレームレートが表示されているよね？
これは、どうやっているの？

Siv3D の Print という機能を使って表示させてるんだ。
Print << hensuu の記述で、文字列や簡単な変数なら表示できるよ。
また、ClearPrint(); で、毎回表示をリセットさせているんだ。

Print は、デバッグ表示にも使えそうだね。

#1-6 : この章のまとめ

今回は、基本的な動作について解説したよ。
System の関数は、他に、強制終了をさせる Exit() などがある。
詳細は、ヘルプで確認してみてね。
